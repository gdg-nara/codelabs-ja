2.2 画像のダウンロード

CardViewはプレーンテキストのみでの使用を意図したものではありません：それはコンテンツの混合物を表示するのに最適です。各行にバナー画像を追加することで、このアプリをよりエキサイティングなものにする良い機会があります。

画像を使用することは、アプリのリソースを消費します。Androidフレームワークは、アプリが画像の小さなサムネイルを表示するだけの場合でも、画像全体をフル解像度でメモリにロードしなければなりません。

このセクションでは、Glide ライブラリを使って大きな画像を効率的に読み込む方法を学びます。

  1. バナー画像のzipファイルをダウンロードしてください。
  2. オペレーティングシステムのファイルエクスプローラでMaterialMe > app > src > main > resディレクトリを開き、描画可能なディレクトリを作成し、個々のグラフィックファイルを描画可能なディレクトリにコピーします。
  3. Sports オブジェクトに含めるためには、各画像へのパスを含む配列が必要です。これを行うには、string.xmlファイル内の項目として、描画可能な画像へのパスをすべて含む配列を定義します。これらの配列は、同じファイルで定義されている sports_titles 配列と同じ順番であることを確認してください。
<array name="sports_images">
   <item>@drawable/img_baseball</item>
   <item>@drawable/img_badminton</item>
   <item>@drawable/img_basketball</item>
   <item>@drawable/img_bowling</item>
   <item>@drawable/img_cycling</item>

2.3 Sportオブジェクトの変更

Sportオブジェクトには、スポーツに対応するDrawableリソースを含める必要があります。これを実現するには

private final int imageResource;
  1. コンストラクタを変更して、整数をパラメータとして受け取り、それをメンバ変数に代入するようにします。
public Sport(String title, String info, int imageResource) {

  1. リソース整数のゲッターを作成します。
public int getImageResource() {

2.4 initializeData() メソッドの修正

MainActivityでは、Sportオブジェクトのコンストラクタが3番目のパラメータとして画像リソースを要求するため、 initializeData()メソッドが壊れてしまいました。

使用する便利なデータ構造は、TypedArrayになります。TypedArrayは、他のXMLリソースの配列を格納することができます。TypedArrayを使用して、同じループ内のインデックスを使用して、画像リソースだけでなく、スポーツのタイトルと情報を取得することができます。

  1. initializeData()メソッドで、getResources().getTypedArray()を呼び出してリソースIDのTypedArrayを取得し、string.xmlファイルで定義したDrawableリソースの配列名を渡します。
TypedArray sportsImageResources = 
        getResources().obtainTypedArray(R.array.sports_images);

配列内のリソースの型に応じて、適切な"get"メソッドを使用してTypedArrayのインデックスiの要素にアクセスすることができます。この特定のケースでは、リソースIDが含まれているので、getResourceId()メソッドを使用します。

  1. スポーツオブジェクトを作成するループのコードを修正し、TypedArray上でgetResourceId()を呼び出すことで、3番目のパラメータとして適切なDrawableリソースIDを追加します。
for(int i=0;i<sportsList.length;i++){
   mSportsData.add(new Sport(sportsList[i],sportsInfo[i], 
       sportsImageResources.getResourceId(i,0)));
}
  1. スポーツデータArrayListを作成したら、型付けされた配列のデータをクリーンアップします。
sportsImageResources.recycle();

2.6 Glideを使って画像を読み込む

画像をダウンロードして ImageView を設定したら、次は onBindViewHolder() で ImageView に画像を読み込むように SportsAdapter を修正します。この方法を取ると、「Out of Memory」エラーでアプリがクラッシュすることに気づくでしょう。Android フレームワークは、ImageView の表示サイズに関係なく、フル解像度で毎回画像をメモリにロードしなければなりません。

画像読み込み時のメモリ消費を減らす方法はいくつかありますが、最も簡単な方法はGlide のような画像読み込みライブラリを使用することです。Glide はバックグラウンド処理やその他の複雑な処理を使用して、画像の読み込み時に必要なメモリを削減します。また、目的の画像が読み込まれている間にプレースホルダ画像を表示するなどの便利な機能も備えています。

注: アプリのメモリ消費量を削減する方法の詳細については、「Large Bitmapsを効率的にロードする」を参照してください。

  1. build.gradle (Module: app) ファイルを開き、依存関係セクションに以下の Glide 用の依存関係を追加します。
implementation 'com.github.bumptech.glide:glide:3.7.0'
  1. SportsAdapterを開き、ImageViewのViewHolderクラスに変数を追加します。
private ImageView mSportsImage;
  1. ViewHolderクラスのViewHolderコンストラクタで変数を初期化します。
mSportsImage = itemView.findViewById(R.id.sportsImage);
  1. ViewHolderクラスのbindTo()メソッドに以下のコードを追加して、Sportオブジェクトから画像リソースを取得し、Glideを使用してImageViewにロードします。
Glide.with(mContext).load(currentSport.getImageResource()).into(mSportsImage);
  1. アプリを実行すると、リストの項目に大胆なグラフィックが表示され、ユーザー体験をダイナミックでエキサイティングなものにしてくれるはずです。

Glideで画像を読み込むのはこれだけです。Glideには他にもいくつかの機能があり、画像のサイズを変更したり、変換したり、様々な方法で画像を読み込むことができます。詳しくはGlideのGitHubページをご覧ください。

ユーザーはアプリでカードを見ると、そのカードがどのように振る舞うのかを期待します。マテリアルデザインでは、以下のガイドラインを提供しています。

  • カードは通常、スワイプで退会させることができます。
  • カードのリストは、カードを長押ししてドラッグすることで並び替えることができます。
  • カードをタップすると詳細が表示されます。

これで、これらの動作をアプリに実装することができました。

3.1 スワイプによる解雇の実装

Android SDKにはItemTouchHelperというクラスがあり、ユーザーがスワイプやドラッグ＆ドロップなどの様々なタッチアクションを行ったときに、RecyclerViewのリスト項目に何が起こるかを定義するために使用されます。一般的なユースケースのいくつかは、ItemTouchHelper.SimpleCallback.ItemTouchHelper.SimpleCallback のメソッドのセットに実装されています。

ItemTouchHelper.SimpleCallbackでは、リスト項目のスワイプや移動でサポートされる方向を定義し、スワイプや移動の動作を実装することができます。

以下のようにしてください。

  1. MainActivityを開いて、最後にあるonCreate()メソッドで、initializeData()メソッドの下に新しいItemTouchHelperオブジェクトを作成します。その引数には、ItemTouchHelper.SimpleCallbackの新しいインスタンスを作成します。新しいItemTouchHelperを入力すると、サジェストが表示されます。サジェストメニューから「ItemTouchHelper.SimpleCallback{...}」を選択します。Android Studioでは、以下のようにonMove()とonSwiped()という必要なメソッドを埋めていきます。
ItemTouchHelper helper = new ItemTouchHelper(new 
                             ItemTouchHelper.SimpleCallback() {
    @Override
    public boolean onMove(RecyclerView recyclerView, 
                    RecyclerView.ViewHolder viewHolder, 
                    RecyclerView.ViewHolder target) {
                return false;
            }

    @Override
    public void onSwiped(RecyclerView.ViewHolder viewHolder, 
                                               int direction) {

    }
});

必要なメソッドが自動的に追加されていない場合は、左マージンの赤い球根をクリックして、メソッドの実装を選択します。

SimpleCallback コンストラクタは赤で下線が引かれていますが、これは必要なパラメータをまだ提供していないからです。

  1. 現時点ではスワイプでの離脱しか実装していないので、対応している移動方向は0、対応しているスワイプ方向はItemTouchHelper.LEFTT | ItemTouchHelper.RIGHTで指定します。
ItemTouchHelper helper = new ItemTouchHelper(new ItemTouchHelper
                .SimpleCallback(0, ItemTouchHelper.LEFT |
                ItemTouchHelper.RIGHT) {
  1. ここで、onSwiped() で希望する動作を実装する必要があります。この場合、カードを左右にスワイプすると、リストからカードが削除されます。remove() をデータセット上で呼び出し、ViewHolder から位置を取得して適切なインデックスを渡します。

mSportsData.remove(viewHolder.getAdapterPosition());

  1. RecyclerViewが削除を適切にアニメーションさせるためには、notifyItemRemoved()を呼び出し、再度ViewHolderから位置を取得して適切なインデックスを渡す必要があります。
mAdapter.notifyItemRemoved(viewHolder.getAdapterPosition());
  1. MainActivityのonCreate()メソッドの新しいItemTouchHelperオブジェクトの下で、ItemTouchHelperインスタンスのattachToRecyclerView()を呼び出して、それをRecyclerViewに追加します。
helper.attachToRecyclerView(mRecyclerView);
  1. あなたのアプリを実行して、あなたは今、それらを削除するには、リストの項目を左右にスワイプすることができます

3.2 ドラッグ＆ドロップの実装

同じ SimpleCallback を使用してドラッグ＆ドロップ機能を実装することもできます。SimpleCallback の最初の引数は、ItemTouchHelper がオブジェクトを移動するためにサポートしている方向を決定します。以下のようにします。

  1. SimpleCallbackの第1引数を0からすべての方向を含むように変更します。
ItemTouchHelper helper = new ItemTouchHelper(new ItemTouchHelper
.SimpleCallback(ItemTouchHelper.LEFT | ItemTouchHelper.RIGHT | 
ItemTouchHelper.DOWN | ItemTouchHelper.UP, ItemTouchHelper.LEFT | ItemTouchHelper.RIGHT) {
  1. onMove()メソッドでは、渡された第2引数と第3引数(元のビューホルダーとターゲットのビューホルダーに対応)から元のインデックスとターゲットのインデックスを取得します。
int from = viewHolder.getAdapterPosition();
int to = target.getAdapterPosition();
  1. Collections.swap()を呼び出してデータセット内の項目を入れ替え、データセットと初期インデックスと最終インデックスを渡します。
Collections.swap(mSportsData, from, to);
  1. 項目が移動されたことをアダプタに通知し、新旧のインデックスを渡し、return 文を true に変更します。
mAdapter.notifyItemMoved(from, to);
return true;
  1. アプリを起動します。リストのアイテムを左右にスワイプして削除したり、長押しで並び替えたりして、ドラッグ＆ドロップモードを有効にすることができるようになりました。

3.3 DetailActivityレイアウトの実装

マテリアルデザインのガイドラインによると、カードはより詳細な情報への入り口を提供するために使用されます。スポーツの詳細情報を見るためにカードをタップしていることに気づくかもしれませんが、それはカードがどのような振る舞いをすることを期待しているからです。

このセクションでは、任意のリスト項目が押されたときに起動される詳細アクティビティを追加します。この実用的な方法として、詳細アクティビティにはクリックしたリスト項目の名前と画像が含まれますが、一般的なプレースホルダの詳細テキストのみが含まれますので、各リスト項目のカスタム詳細を作成する必要はありません。

  1. ファイル＞新規作成＞アクティビティ＞空のアクティビティと進み、新しいアクティビティを作成します。
  2. これを DetailActivity と呼び、すべてのデフォルトを受け入れます。
  3. 新しく作成したactivity_detail.xmlレイアウトファイルを開き、以前の演習で行ったように、ルートViewGroupをRelativeLayoutに変更します。
  4. RelativeLayout から xmlns:app="http://schemas.android.com/apk/res-auto" ステートメントを削除します。
  5. list_item.xmlファイルからTextViewとImageViewのすべての要素をactivity_detail.xmlファイルにコピーします。
  6. list_item.xmlのIDと区別するために、各android:id属性の参照先に"Detail"という単語を追加します。例えば、ImageView IDをsportsImageからsportsImageDetailに変更します。
  7. すべてのTextViewとImageView要素において、layout_belowのような相対配置のためのIDへのすべての参照を"詳細"IDを使用するように変更します。
  8. subTitleDetail TextViewでは、プレースホルダテキスト文字列を削除し、詳細テキストの代わりに一般的なテキストの段落を貼り付けます（例えば、Lorem Ipsumの数段落）。テキストを文字列リソースに抽出します。
  9. TextView要素のパディングを16dpに変更する。
  10. RelativeLayout 全体を ScrollView でラップします。必要な layout_height と layout_width 属性を追加し、XMLNS:android="http://schemas.android.com/apk/res/android" 属性を ScrollView の最後に追加します。
  11. RelativeLayoutのlayout_height属性を"wrap_content"に変更します。

activity_detail.xmlレイアウトの最初の2つの要素は、次のようになります。

<?xml version="1.0" encoding="utf-8"?>
<ScrollView xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <RelativeLayout xmlns:tools="http://schemas.android.com/tools"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        tools:context="com.example.android.materialme.DetailActivity">

4.2 マテリアルデザインパレットの選択

Material Designでは、アプリのために少なくともこの3色を選ぶことをお勧めしています。

  • 原色です。これは、アプリバー（アプリのタイトルを含むバー）の色に自動的に使用されます。
  • 原色の濃い色。同じ色の濃い色合いのこと。特にアプリバーの上のステータスバーなどに使われています。
  • アクセントカラー。原色とのコントラストが良い色。様々なハイライトに使われますが、FABのデフォルトカラーでもあります。

アプリを実行したときに、FAB色とアプリバーの色がすでに設定されていることに気づいたかもしれません。このタスクでは、これらの色がどこに設定されているかを学びます。マテリアルカラーガイドを使用して、いくつかの色を選んで実験することができます。

  1. Project > Android ペインで、styles.xml ファイル（values ディレクトリにあります）に移動します。AppThemeスタイルでは、デフォルトで3つの色が定義されています：colorPrimary、colorPrimaryDark、およびcolorAccent。これらのスタイルは、colors.xmlファイルの値によって定義されます。
  2. そのような＃607D8B（ブルーグレーの色見本で）のようなあなたの原色として使用するために、材料の色ガイドから色を選択します。あなたはまだ適切なアクセントと暗い色を選ぶことができるように、それは色見本の300から700の範囲内でなければなりません。
  3. colors.xmlファイルを開き、選択した色に合わせてcolorPrimaryの16進数値を変更します。
  4. 暗い色の原色として使用するために、同じ色の暗い色調を選びます。ここでも、colorPrimaryDarkのcolors.xmlの16進数値を変更して一致させます。
  5. 値がAで始まり、原色とのコントラストが良い色（ディープオレンジA200など）から、FABのアクセントカラーを選択します。colors.xmlのcolorAccentの値を変更して一致させます。
  6. アプリを実行します。アプリバーとFABが新しいカラーパレットを反映して変更されました

FABの色をテーマカラー以外に変更したい場合は、app:backgroundTint属性を使用します。これは app: 名前空間を使用し、Android Studio は名前空間を定義するためのステートメントを追加するように求めてくることに注意してください。
