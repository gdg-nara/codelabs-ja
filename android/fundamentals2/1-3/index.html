導入

TextView クラスは、画面上にテキストを表示する View クラスのサブクラスです。テキストの表示方法は、XML レイアウトファイルの TextView 属性で制御することができます。この実習では、複数の TextView 要素を扱う方法を紹介します。その中には、ユーザーが内容を縦にスクロールできるものもあります。

デバイスのディスプレイに収まりきらないほどの情報がある場合、スクロールビューを作成して、ユーザーが上下にスワイプすることで垂直方向に、左右にスワイプすることで水平方向にスクロールできるようにすることができます。

スクロールビューは、ニュースや記事など、ディスプレイに収まりきらない長さのテキストによく使われます。また、ユーザーが複数行のテキストを入力できるようにしたり、スクロールビューの中で UI 要素（テキストフィールドとボタンなど）を組み合わせたりする場合にも、スクロールビューを使用します。

ScrollView クラスは、スクロール・ビューのレイアウトを提供します。ScrollView は、FrameLayout のサブクラスです。その中に子として1つのビューだけを配置します。子View は、スクロールするコンテンツ全体を含んでいます。この子 View は、それ自体が UI 要素を含むViewGroup（LinearLayout など）である可能性があります。

複雑なレイアウトでは、画像などの子View でパフォーマンスの問題が発生する可能性があります。ScrollView 内の View としては、ユーザーがスクロールできるアイテム(TextView 要素など)を表示する、垂直方向に配置された LinearLayout が適しています。

ScrollView では、画面に表示されていなくても、すべてのUI要素がメモリ上にあり、ビュー階層にあります。ScrollView は、テキストがすでにメモリ内にあるため、自由形式のテキストのページをスムーズにスクロールさせるのに適しています。しかし、ScrollView は多くのメモリを使用するため、アプリの他の部分のパフォーマンスに影響を与える可能性があります。ユーザーが追加、削除、編集できるアイテムの長いリストを表示するには、 RecyclerView の使用を検討してください。

1.2 記事のテキストを追加する

雑誌や新聞の記事にアクセスする実際のアプリでは、表示される記事はおそらくコンテンツプロバイダを通じてオンラインソースから送られてくるか、あるいはデバイス上のデータベースにあらかじめ保存されているかもしれません。

今回の実用では、記事を string.xml リソースで1つの長い文字列として作成します。

  1. app > res > values フォルダで、strings.xml を開きます。
  2. 文字数の多いテキストファイルを開くか、完成したScrollingTextアプリのstrings.xmlファイル を開きます。
  3. article_title と article_subtitle の文字列の値は、タイトルとサブタイトルを作って入力するか、ScrollingText アプリの完成品の strings.xml ファイルの値を使用してください。文字列の値は、HTML タグや複数行を含まない1行のテキストにしてください。
  4. article_text には、文字列を入力するか、コピー＆ペーストしてください。

テキストファイルのテキストを使用するか、完成した ScrollingText アプリの strings.xml ファイルの article_text 文字列に提供されているテキストを使用します。このタスクの唯一の要件は、テキストが画面に収まらないほどの長さであることです。

以下の点に注意してください（例として下図を参照してください）。

  • strings.xml ファイルにテキストを入力したり貼り付けたりすると、テキストの行が次の行に折り返されず、右マージンからはみ出してしまいます。これは正しい動作で、左マージンから始まるテキストの各行が段落全体を表しています。strings.xml のテキストを折り返して表示したい場合は、Return キーを押して改行コードを入力するか、改行コードを保存するテキストエディタで最初にテキストをフォーマットします。
  • 改行を表すには行末に \n を、空行を表すには \n のみを入力します。段落同士がぶつからないようにするためには、行末の文字を追加する必要があります。
  • 文章中にアポストロフィ（'）がある場合は、アポストロフィの前にバックスラッシュ（\）を付けてエスケープする必要があります。文章中にダブルクオートがある場合も、それをエスケープしなければなりません（"）。その他の非ASCII文字もエスケープする必要があります。詳細は、文字列リソースの フォーマットとスタイル のセクションを参照してください。
  • 太字にしたい場合は、HTML の <b> および </b> タグを入力してください。
  • イタリック体にしたい場合は、HTML の <i> および </i> タグを入力してください。イタリック体のフレーズの中でカールしたアポストロフィーを使用している場合は、ストレートなアポストロフィー（'）に置き換えてください。（補足：英語文化圏では、引用符にカールしたアポストロフィーを使います）
  • <b><i>...単語...</i></b> のように、タグを組み合わせることで、ボールドとイタリックを組み合わせることができます。その他の HTML タグは無視されます。
  • strings.xml ファイル内の <string name="article_text"> </string> でテキスト全体を囲みます。
  • www.google.com のように、テスト用の Web リンクを含める。(以下の例では、www.rockument.com を使用しています。) HTML タグは使用しないでください。太字と斜体のタグ以外の HTML タグは無視され、テキストとして表示されます。

2.1 アクティブなWebリンクのためのautoLink属性の追加

記事を表す article の TextView に android:autoLink="web" 属性を追加します。このTextView の XML コードは次のようになります。

<TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:id="@+id/article"
        android:autoLink="web"
        android:layout_below="@id/article_subheading"
        android:lineSpacingExtra="@dimen/line_spacing"
        android:padding="@dimen/padding_regular"
        android:text="@string/article_text" />

2.2 ScrollViewのレイアウトへの追加

View（TextViewなど）をスクロール可能にするには、ScrollView の中に View を埋め込みます。

  1. article_subheading TextView と article TextView の間に ScrollView を追加します。<ScrollView>と入力すると、Android Studio は自動的に最後に</ScrollView>を追加し、android:layout_widthとandroid:layout_height 属性の候補を表示します。
  2. 両方の属性の候補から wrap_content を選びます。

これで、2 つの TextView 要素と ScrollView のコードは次のようになりました。

<TextView
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:id="@+id/article_subheading"
   android:layout_below="@id/article_heading"
   android:padding="@dimen/padding_regular"
   android:text="@string/article_subtitle"
        android:textAppearance=
                     "@android:style/TextAppearance.DeviceDefault"/>

<ScrollView
   android:layout_width="wrap_content"
   android:layout_height="wrap_content"></ScrollView>

<TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:id="@+id/article"
        android:autoLink="web"
        android:layout_below="@id/article_subheading"
        android:lineSpacingExtra="@dimen/line_spacing"
        android:padding="@dimen/padding_regular"
        android:text="@string/article_text" />
  1. 末尾の </ScrollView> コードを article TextView の後に移動し、article TextView の属性が全て ScrollView の中に入るようにします。
  2. 次の属性を article TextView から削除し、ScrollView に追加します。
android:layout_below="@id/article_subheading"

上記の属性を設定すると、ScrollView 要素は記事の小見出しの下に表示されます。記事はScrollView 要素の中にあります。

  1. Code > Reformat Code を選択してXMLコードを再フォーマットし、article TextView が <ScrollView> コードの中でインデントされて表示されるようにします。
  2. レイアウトエディタの右側にある Preview タブをクリックすると、レイアウトのプレビューが表示されます。

</RelativeLayout>

前述の通り、ScrollView は1つの子 View(作成した記事の article TextView など)しか含むことができません。しかし、その View は、LinearLayout のような View 要素を含む別の ViewGroup にすることができます。ScrollView の中に LinearLayout などの ViewGroup を入れ子にすることで、LinearLayout の中にある全てのものをスクロールすることができます。

例えば、記事の小見出しを記事と一緒にスクロールさせたい場合は、ScrollView 内に LinearLayout を追加し、小見出しと記事を LinearLayout 内に移動させます。下図のように、LinearLayout は ScrollView の単一の子 View となり、ユーザーは LinearLayout 全体（小見出しと記事）をスクロールすることができます。

3.1 ScrollView に LinearLayout を追加する

  1. ScrollingText アプリプロジェクトの activity_main.xml ファイルを開き、Text タブを選択して(まだ選択されていない場合) 、XML コードを編集します。
  2. ScrollView の中の記事の article TextView の上に LinearLayout を追加します。<LinearLayout> と入力すると、Android Studio は自動的に末尾に </LinearLayout> を追加し、android:layout_width と android:layout_height 属性の候補を提示します。その幅と高さの候補から、それぞれ match_parent と wrap_content を選びます。ScrollView の最初のコードは次のようになります。
<ScrollView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@id/article_subheading">
        
        <LinearLayout
            android:layout_width="match_parent"
            android:layout_height="wrap_content"></LinearLayout>

        <TextView
            android:id="@+id/article"

match_parent を使用して、親 ViewGroup の幅に合わせます。wrap_content を使用して LinearLayoutのサイズを変更し、コンテンツを囲むのに十分な大きさにします。

  1. 最後の </LinearLayout> コードを、記事の article TextView の後、</ScrollView> の前に移動します。

これで LinearLayout は記事の article TextView を含み、完全に ScrollView の中に入りました。

  1. LinearLayout に android:orientation="vertica" 属性を追加して、向きを垂直にします。
  2. Code > Reformat Code を選択して、コードを正しくインデントづけします。

これで LinearLayout は次のようになります : 

<ScrollView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@id/article_subheading">

        <LinearLayout
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:orientation="vertical">

            <TextView
                android:id="@+id/article"
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:autoLink="web"
                android:lineSpacingExtra="@dimen/line_spacing"
                android:padding="@dimen/padding_regular"
                android:text="@string/article_text" />
        </LinearLayout>

</ScrollView>

3.2 LinearLayout 内での UI 要素の移動

LinearLayout は今、1 つのUI要素として記事の article TextView だけを持っています。article_subheading TextView を LinearLayout の中に入れて、両方がスクロールするようにしたいと思います。

  1. article_subheading TextView を移動するには、コードを選択し、Edit > Cut を選択し、LinearLayout 内の article TextView の上をクリックして、Edit > Paste を選択します。
  2. article_subheading TextView から android:layout_below="@id/article_heading" 属性を削除します。この TextView は LinearLayout の中に入っているので、この属性は LinearLayout の属性と衝突してしまいます。
  3. ScrollView の layout 属性を android:layout_below="@id/article_subheading" からandroid:layout_below="@id/article_heading" に変更します。小見出しが LinearLayout の一部になったことで、ScrollView は小見出しではなく、見出しの下に配置しなければなりません。

ScrollView の XML コードは以下のようになります : 

<ScrollView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@id/article_heading">

        <LinearLayout
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:orientation="vertical">

            <TextView
                android:id="@+id/article_subheading"
                android:layout_width="match_parent"
                android:layout_height="wrap_content"
                android:padding="@dimen/padding_regular"
                android:text="@string/article_subtitle"
                android:textAppearance=
                     "@android:style/TextAppearance.DeviceDefault" />

            <TextView
                android:id="@+id/article"
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:autoLink="web"
                android:lineSpacingExtra="@dimen/line_spacing"
                android:padding="@dimen/padding_regular"
                android:text="@string/article_text" />
        </LinearLayout>

</ScrollView>
  1. アプリを起動します。

上下にスワイプして記事をスクロールさせると、小見出しが記事と一緒にスクロールし、見出しは固定されていることに気づきます。

課題: ScrollView 内の LinearLayout に別の UI 要素（Button）を追加し、テキストと一緒にスクロールさせるようにしてください。

  • Buttonには Add Comment というテキストを使用します。この課題では、Buttonを処理するメソッド(button-handling method) を作成する必要はありません。すべきことは、Button 要素をレイアウトの適切な場所に配置することだけです。
