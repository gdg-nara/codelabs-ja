導入

1.2 パート A: 初めてのインタラクティブな UI で学んだように、レイアウトエディタの ConstraintLayout を使ってユーザーインターフェース (UI) を構築することができます。このレイアウトエディタは、他の要素やレイアウトのエッジへの制約接続を使って UI 要素をレイアウトに配置します。ConstraintLayout は、UI要素 をレイアウト・エディタに簡単にドラッグできるように設計されています。

ConstraintLayoutは、他のViewオブジェクト(childrenまたはchild viewと呼ばれる)を含むことができる特別なViewである ViewGroupです。この実習では、ConstraintLayoutとレイアウトエディタのより多くの機能を紹介します。

この実習では、他にも2つのViewGroupサブクラスを紹介しています。

  • LinearLayout（線形レイアウト）：グループ内の子View要素を水平または垂直に整列させるもの。
  • RelativeLayout（相対レイアウト）：子View要素のグループで、各View要素がViewGroup内の他のView要素に対して相対的に配置されています。子View要素の位置は、相互または親 ViewGroup との関係で記述されます。

1.2 横向き用のレイアウトバリアントを作る

このレイアウトの垂直方向と水平方向の見た目の違いは、show_count TextView要素の数字(0)が水平方向では低すぎて、Countボタンに近すぎることです。お使いのデバイスやエミュレータによっては、テキストサイズが160spに固定されているため、TextView要素が大きすぎたり、中央に表示されないことがあります。

縦方向はそのままにして横方向だけを修正するには、Hello Toastアプリのレイアウトを横方向で異なるものにするとよいでしょう。以下の手順で行います。

  1. 上部ツールバーの「画面の向きとレイアウトバリエーション」ボタンをクリックします。
  2. 「Create Landscape Variation」を選択します。

新しいエディターウィンドウが開き、land/activity_main.xmlタブに横向き（水平）方向のレイアウトが表示されます。この横向き専用のレイアウトは、元々の縦向き（垂直）を変えずに変更することができます。

  1. 「Projectt > Android」ペインで「res > layout」ディレクトリの中を見ると、Android Studioが自動的に「activity_main.xml（land）」というバリアントを作成してくれていることがわかります。

1.5 タブレット用のレイアウトバリエーションを作る

前回学んだように、上部ツールバーの「デバイスタイプとサイズ」ボタンをクリックすると、さまざまなデバイス用のレイアウトをプレビューすることができます。メニューからNexus 10（タブレット）などのデバイスを選ぶと、タブレットの画面には理想的なレイアウトになっていないことがわかります。各Buttonのテキストは小さすぎ、上下のButton要素の配置も大画面のタブレットには適していません。

この問題をタブレット向けに解決するためには、携帯電話サイズの横方向と縦方向はそのままにして、タブレット向けに完全に異なるレイアウトのバリエーションを作成します。以下の手順で作成してください。

  1. 「Design（デザイン）」タブをクリックすると（まだ選択されていない場合）、「デザイン」と「ブループリント」のペインが表示されます。
  2. 上部ツールバーの「画面の向きとレイアウトバリエーション」ボタンをクリックします。
  3. Create layout x-large Variation を選択します。

新しいエディターウィンドウが開き、xlarge/activity_main.xml タブにタブレットサイズのデバイス用のレイアウトが表示されます。エディターでは、Nexus 9やNexus 10などのタブレット端末もプレビュー用に選べます。タブレット専用のこのレイアウトは、他のレイアウトを変更することなく変更することができます。

1.6 タブレット端末向けにレイアウトのバリエーションを変更

「Design（デザイン）」タブの「Attributes（属性）」パネルを使って、このレイアウトの属性を変更することができます。

  1. ツールバーの「親への自動接続」ツール機能をオフにします。この手順では、ツールが無効になっていることを確認してください。
  2. ツールバーの「制約の全削除」ボタンをクリックして、レイアウトのすべての制約を解除します。

制約がなくなったことで、レイアウト上の要素の移動やサイズ変更が自由にできるようになりました。

  1. レイアウトエディターでは、要素の四隅にリサイズハンドルがあり、サイズを変更することができます。Component Tree（コンポーネントツリー）で、show_countというTextViewを選択します。TextViewが邪魔にならないようにして、Button要素を自由にドラッグできるようにするには、下図のように、TextViewの角をドラッグしてサイズを変更します。

要素のサイズを変更すると、幅と高さの寸法がハードコードされます。ハードコードされた寸法が、さまざまなサイズや密度の画面上でどのように見えるかは予測できないため、ほとんどの要素でサイズ寸法をハードコードすることは避けてください。現在は、要素を邪魔にならないように移動させるためにこの操作を行っていますが、寸法の変更は別のステップで行います。

  1. コンポーネントツリーで button_toast Buttonを選択し、「Attributes（属性）」タブをクリックして「属性」ペインを開き、textSizeを60spに変更し（下図の1）、layout_widthをwrap_contentに変更します（下図の2）。

上の図（2）の右側にあるように、正方形の左右に2つのセグメントで表示されるビューインスペクタの幅コントロールを、「Wrap Content」と表示されるまでクリックします。別の方法として、layout_widthメニューからwrap_contentを選択することもできます。

  1. アニメーション図のように、button_toast Buttonをレイアウトの上部と左側に拘束し、button_count Buttonをbutton_toast Buttonの近くのスペースにドラッグして、button_count Buttonの左側をbutton_toast Button に拘束します。

  1. ベースライン制約を使って、button_count Buttonのテキストベースラインがbutton_toast Buttonのテキストベースラインと一致するように制約をかけることができます。button_count要素を選択し、要素の下にベースライン制約ボタンが表示されるまで、ポインタを要素の上に置いてください。
  2. ベースライン制約ボタンをクリックします。ベースラインのハンドルが表示され、アニメーション図のように緑色に点滅します。ベースライン制約線をクリックして、button_toast要素のベースラインにドラッグします。

LinearLayout は、ビューのコレクションを水平または垂直に並べる ViewGroup です。

LinearLayoutは、シンプルで高速なため、最も一般的なレイアウトの1つです。他のビューグループの中で、UI要素を水平または垂直に配置するためによく使用されます。

LinearLayoutにはこれらの属性が必要です。

  • layout_width
  • layout_height
  • orientation

layout_widthおよびlayout_heightは、これらの値のいずれかを取ることができます。

  • match_parent: 親ビューの幅または高さに合わせてビューを拡大します。LinearLayoutがルートビューの場合は、画面（親ビュー）のサイズに合わせて拡大します。
  • wrap_content: ビューの寸法を縮小して、ビューがコンテンツを囲むのに十分な大きさにします。コンテンツがない場合、ビューは見えなくなります。
  • 固定のdp ( 密度非依存ピクセル) : デバイスの画面密度に合わせて調整した固定サイズを指定します。例えば、16dpは密度に依存しない16個のピクセルを意味します。

オリエンテーションができる。

  • horizontal**:** ビューは左から右に向かって配置されます。
  • vertical**:** ビューは上から下に向かって配置されます。

このタスクでは、Hello ToastアプリのConstraintLayoutルートビューグループをLinearLayoutに変更し、LinearLayoutの使用方法を習得します。

2.1 ルートビューグループをLinearLayoutに変更する

  1. 前述のタスクで使用したHello Toastアプリを開きます。
  2. activity_main.xmlレイアウトファイルを開き（まだ開いていない場合）編集ペインの下部にあるTextタブをクリックしてXMLコードを表示します。XMLコードの最上部には、次のようなタグラインがあります。
<android.support.constraint.ConstraintLayout xmlns:android="http:..."。
  1. 「<android.support.constraint.ConstraintLayout 」タグを「 <LinearLayout 」に変更し、以下のようなコードにします。

ConstraintLayoutの使用:

  • ConstraintLayoutルートを持つレイアウトのすべての制約をクリアするには、ツールバーの「制約の全削除」ボタンをクリックします。
  • TextViewやButtonなどのテキストを含むUI要素を、テキストを含む別のUI要素に整列させることができます。ベースライン制約では、テキストのベースラインが一致するように要素を制約することができます。
  • ベースライン制約を作成するには、要素の下に「ベースライン」ボタンが表示れるまで、UI要素の上にポインタを置きます。
  • ツールバーの「パック」ボタンには、選択したUI要素をパックまたは拡張するためのオプションが用意されています。これを使って、Button 要素をレイアウト上で水平方向に均等に配置することができます。

LinearLayoutの使用:

  • LinearLayoutは、ビューのコレクションを水平または垂直に並べるViewGroup です。
  • LinearLayoutには、layout_width、layout_height、orientationの各属性が必要です。
  • match_parent で layout_width または layout_height の指定: View をその親の幅または高さに合わせて拡張します。LinearLayoutがルートViewの場合、画面（親View）のサイズに合わせて拡大します。
  • layout_widthまたはlayout_heightのWrap_content: Viewがコンテンツを囲むのに十分な大きさになるように寸法を縮小します。コンテンツがない場合、Viewは見えなくなります。
  • layout_width または layout_height に固定のdp ( 密度非依存ピクセル) の指定: デバイスの画面密度に合わせて調整した固定サイズを指定します。例えば、16dpは密度に依存しない16個のピクセルを意味します。
  • LinearLayoutの向きは、要素を左から右に並べるhorizontal（水平方向）と、要素を上から下に並べるvertical（垂直方向）があります。
  • gravityとweight属性を指定することで、LinearLayout内のビューやコンテンツの配置をさらにコントロールすることができます。
  • android:gravity属性は、ViewのコンテンツがView自体の中でどのように配置されるかを指定します。
  • android:layout_weight属性は、LinearLayoutの余分なスペースのうち、どれだけをViewに割り当てるかを示しています。1つのViewだけがこの属性を持っている場合、そのViewがすべての余分な画面スペースを取得します。複数の View 要素の場合、スペースは比例配分されます。例えば、2つのButton要素がそれぞれ1のウェイトを持ち、TextViewが2のウェイトを持ち、合計4の場合、Button要素はそれぞれのスペースの1/4を得て、TextViewは半分を得ます。

RelativeLayoutの使用:

  • RelativeLayoutは、各ビューがグループ内の他のビューに対して相対的に配置されている ViewGroupです。
  • Viewを親の上に合わせるには、android:layout_alignParentTopを使います。
  • Viewを親の左側に揃えるには、android:layout_alignParentLeftを使います。
  • android:layout_alignParentStartを使うと、Viewの開始エッジが親の開始エッジと一致するようになります。この属性は、異なる言語やロケールの環境設定を使用しているデバイスでアプリを動作させたい場合に便利です。開始は、左から右への環境設定の場合は画面の左端、右から左への環境設定の場合は画面の右端となります。
